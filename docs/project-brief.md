# Project Brief

## Working Title
Recycle Rush

## Core Idea
A public website that helps users build recycling awareness through a playful drag-and-drop sorting game.

## Primary Goal
Teach people how to sort waste correctly by letting them move flying waste items into the correct recycling bins.

## Core Gameplay
- Waste items move across the screen.
- A user can click or drag an item.
- The user drops the item into one of four bins: general waste, paper, plastic, and organic waste.
- A correct drop adds 1 point.
- An incorrect drop triggers a red full-screen flash and a sad smiley reaction.
- An incorrect drop resets the score to 0 and the user starts the run again.
- For the MVP, wrong-drop feedback only communicates that the action was wrong and does not reveal the correct bin.
- The score persists only for the current browser session while the page remains open.
- Refreshing or reopening the site resets the score to 0.
- Each round lasts 60 seconds.

## Product Constraints
- No login or account system.
- Public access for anyone with the link.
- Designed to be shared with a team.
- Focus on fun, fast interaction and awareness-building.

## Initial User Experience Goals
- Easy to understand within a few seconds.
- Visually lively and playful.
- Immediate feedback on correct and incorrect sorting.
- Desktop-first for the MVP, with mobile support considered later.

## MVP Scope
- Landing/game screen with animated waste items.
- Four recycling bins: general waste, paper, plastic, and organic waste.
- Drag-and-drop interaction.
- Session-only score counter.
- 60-second gameplay round.
- Basic feedback for correct actions.
- Full-screen error feedback and score reset for incorrect actions.
- Teaching feedback appears only the first time a specific item is sorted correctly during a run.
- Reset by refresh behavior.

## Out of Scope For MVP
- Authentication.
- User profiles.
- Persistent high scores.
- Multiplayer competition.
- Admin dashboard.

## Success Criteria
- A new user can understand the objective without instructions or after a very short prompt.
- The game feels responsive and fun.
- Users learn at least basic sorting categories from repeated play.
- The score behaves exactly as expected for a temporary session.
- A wrong answer clearly communicates failure and immediately restarts the current score run.
